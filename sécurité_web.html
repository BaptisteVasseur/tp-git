Sécurité Web : XSS, CSRF & Bonnes Pratiques

Qu'est-ce que la sécurité web ?

La sécurité web vise à protéger les sites et applications contre les attaques et vulnérabilités courantes. Deux des menaces majeures sont les attaques XSS (Cross-Site Scripting) et CSRF (Cross-Site Request Forgery).

1. XSS (Cross-Site Scripting)

XSS permet à un attaquant d'injecter du code JavaScript malveillant dans une page web vue par d'autres utilisateurs.

Exemple d'attaque XSS :

<script>alert('XSS')</script>

Comment s'en protéger ?

  • ✔ Toujours échapper les données affichées (HTML escape).
  • ✔ Utiliser des frameworks qui protègent automatiquement (React, Angular, etc.).
  • ✔ Désactiver l'exécution de scripts dans les champs utilisateurs.
  • ✔ Mettre en place une politique CSP (Content Security Policy).

2. CSRF (Cross-Site Request Forgery)

CSRF consiste à tromper un utilisateur authentifié pour qu'il exécute une action non désirée sur un site où il est connecté.

Exemple d'attaque CSRF :

<img src="https://votresite.com/delete?id=123" />

Comment s'en protéger ?

  • ✔ Utiliser des tokens CSRF dans les formulaires.
  • ✔ Vérifier l'origine (Origin, Referer).
  • ✔ Privilégier les requêtes POST pour les actions sensibles.
  • ✔ Mettre en place une authentification forte.

3. Bonnes pratiques générales

  • ✔ Garder les logiciels et dépendances à jour.
  • ✔ Utiliser HTTPS partout.
  • ✔ Limiter les droits des utilisateurs et des applications.
  • ✔ Valider et filtrer toutes les entrées utilisateur côté serveur.
  • ✔ Sauvegarder régulièrement les données.
  • ✔ Désactiver les messages d’erreur détaillés en production.
  • ✔ Utiliser des mots de passe forts et le stockage sécurisé (hachage, salage).
